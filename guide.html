Angular → Spring Boot → PostgreSQL

How to send signup form data from the frontend, through an API, and save it to the database

Angular Form → AuthService (HttpClient) → HTTP POST Request → Spring Boot Controller → Repository → PostgreSQL (Neon)
1

Fix pom.xml — Remove wrong scope from spring-boot-starter-web

Backend
Currently spring-boot-starter-web has <scope>test</scope> which means it only exists during testing — your REST controllers will not work at all. Remove that scope line.
pom.xmlChange
1<dependency>
2 <groupId>org.springframework.boot</groupId>Maven group for all Spring Boot libs
3 <artifactId>spring-boot-starter-web</artifactId>Brings in REST, HTTP server (Tomcat), JSON support
4 <!-- DELETE the line below -->This line is the bug — it hides the library from production
5 <scope>test</scope>DELETE THIS LINE — scope=test means only available during tests
6</dependency>

Create User.java — The database table blueprint

Backend
This class tells Hibernate (the ORM) what the users table should look like. Each field becomes a column. You do not write SQL — Hibernate auto-creates the table because ddl-auto=update is set in your properties.
src/main/java/com/example/demo/model/User.javaNew File
1package com.example.demo.model;Declares which folder/package this file lives in
3import jakarta.persistence.*;Imports all JPA annotations like @Entity, @Id, @Column
5@EntityTells Hibernate: this class = a database table
6@Table(name = "users")Sets the actual table name in PostgreSQL to "users"
7public class User {
9 @IdMarks this field as the PRIMARY KEY of the table
10 @GeneratedValue(strategy = GenerationType.IDENTITY)Auto-increment the ID — PostgreSQL assigns 1, 2, 3... automatically
11 private Long id;The ID column — Long because IDs can get very large
13 private String name;Becomes the "name" column — stores the user's name
15 @Column(unique = true)Prevents two users from registering with the same email
16 private String email;Becomes the "email" column
18 private String password;Becomes the "password" column
20 // add getters and setters for all fieldsSpring needs getters/setters to read and write JSON automatically
21}
💡 In IntelliJ, right-click inside the class → Generate → Getter and Setter to create all getters/setters automatically.

Create UserRepository.java — Database operations for free

Backend
By extending JpaRepository, Spring automatically gives you methods like save(), findAll(), deleteById() — you write zero SQL.
src/main/java/com/example/demo/repository/UserRepository.javaNew File
1package com.example.demo.repository;Package declaration for this file
3import com.example.demo.model.User;Brings in your User class so this repo knows what to work with

Backend
This is the entry point for HTTP requests from Angular. It listens for a POST to /api/users/signup, receives the JSON body, saves it, and returns the saved user.

Create auth.service.ts — The HTTP call to the backend

Frontend
A service is a reusable class that handles data operations. Keeping the HTTP call here (not in the component) means if the API URL changes, you fix it in one place.
src/app/auth.service.tsNew File
1import { Injectable } from '@angular/core';Brings in the @Injectable decorator
